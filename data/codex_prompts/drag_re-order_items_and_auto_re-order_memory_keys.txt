### 4. Memory key reorder logic

This is the critical part.

When reordering within the same parent list:
- all sibling `memory_key` values under that parent must be recalculated so the ordering remains contiguous and deterministic

Example:
- Parent list has items ordered by `memory_key`
- If an item with `memory_key = 322` is dragged so it should appear before the item with `memory_key = 108`
- The dragged item should become `memory_key = 108`
- The old 108 becomes 109
- 109 becomes 110
- and so on until the moved item's old position is removed from the sequence

I want this implemented safely and consistently.

Please do NOT do fragile one-off updates.

Recommended implementation:
- Fetch all siblings for the parent, ordered by `memory_key`
- Remove the dragged item from the array
- Insert it at the target index
- Reassign `memory_key` values sequentially for the entire sibling set
- Persist the full reordered set in a transaction / safe server-side operation
